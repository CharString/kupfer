Cómo configurar los complementos

La funcionalidad en Kupfer está organizada por módulos de extensiones llamados «complementos». Cada complemento proporciona funcionalidad adicional, por ejemplo, integración con una aplicación externa.

Configurar complementos

Abra las Preferencias de Kupfer y vaya a la pestaña Complementos.

  1. Use uno de los siguientes métodos:

    • Pulse en el icono de Kupfer en el área de notificación y seleccione el elemento Preferencias.

    • Busque el objetoPreferencias de Kupfer en el propio Kupfer. Pulse Intro para abrirlo.

    • Use el atajo de teclado Ctrl+;

  2. Seleccione la solapa Complementos.

Seleccione complementos en la lista para leer sobre ellos, y marque la casilla junto al nombre para activar el complemento, o desmárquela para desactivarlo.

Si el complemento tiene parámetros configurables, estarán visibles debajo de la información del complemento.

El complemento Complementos de Kupfer permite un acceso rápido a la información de cada complemento así como la opción «Mostrar código fuente», que muestra su implementación.

Si no se puede activar un complemento

Si un complemento falla al activarlo porque requiere un módulo de software que no está disponible, la información del complemento mostrará un mensaje como este:

No se pudo leer el complemento debido a un error:

Se necesita el módulo de Python «gdata»

Esto significa que debe instalar el módulo de Python requerido para su distribución; y posiblemente la documentación del complemento le diga cómo hacerlo.

El complemento también puede fallar inesperadamente al cargarse, y mostrar un mensaje de error diferente. Puede ser un error de programación, tanto en el complemento como en Kupfer.

Instalar más complementos

Puede instalar complementos personalizados en la carpeta ~/.local/share/kupfer/plugins. Cada complemento puede ser un único archivo .py o un paquete de Python (una carpeta que directamente contiene un archivo llamado __init__.py). Los complementos en forma de paquete pueden incluir archivos de iconos. Los paquetes de Python también se pueden instalar como archivos .zip.

Precaución: considere los complementos como programas. No instale complementos que no sean de confianza.

Crear complementos

La documentación para creadores de complementos está disponible en el archivo Documentation/Manual.rst en la distribución fuente o en la página web Manual de Kupfer. Una manera sencilla de empezar es copiar un complemento existente y experimentar con él.

La pestaña Catálogo en «Preferencias»

Cada complemento puede exportar un número de fuentes que contienen objetos. Normalmente, todos estos objetos son accesibles directamente desde un nivel de búsqueda superior. Algunos complementos exportan muchos objetos o muy especializados que sus catálogos no debería exportar al nivel superior. Para alcanzar esos objetos, primero debe encontrar el catálogo por nombre, y entrar en el catálogo usando la acción Buscar contenido.

En la pestaña Catálogo en las Preferencias de Kupfer, una casilla marcada junto a cada fuente indica que ese objeto está exportado. Una casilla desmarcada indica que su contenido está oculto desde el nivel superior.

Nota: Kupfer puede ralentizarse si se exportar muchos catálogos grandes al nivel superior.
